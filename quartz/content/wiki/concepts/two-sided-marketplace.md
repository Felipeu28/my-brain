# Two-Sided Marketplace Dynamics

**Type:** Concept — business model framework
**Last updated:** 2026-04-12
**Related:** [[wiki/concepts/network-effects-hiring]], [[wiki/community/blue-collar-networks]], [[wiki/concepts/voice-first-hiring]]

---

## Summary

A two-sided marketplace connects two distinct user groups who create value for each other. Moil is a two-sided marketplace: **employers** (demand side) need to hire workers; **workers** (supply side) need jobs. Neither side is useful without the other.

## The Classic Chicken-and-Egg Problem

Every two-sided marketplace faces the same founding challenge: employers won't use the platform if there are no workers; workers won't use the platform if there are no jobs.

**Moil's solution**: Start with employers. Employers post jobs → jobs are distributed to 10+ external platforms (Indeed, ZipRecruiter, Spanish job groups) → workers apply from those platforms, entering Moil's system for the first time. The employer-first model means Moil doesn't need to build worker supply before it can be useful.

## The Hard Side vs. Easy Side

| Side | Moil | Why |
|------|------|-----|
| **Hard side** | Workers (supply) | Must find them where they are; language and tech barriers |
| **Easy side** | Employers (demand) | They actively want to hire; ROI is clear; they'll pay |

Moil's voice-first, Spanish-first approach is fundamentally a **supply-side strategy**: remove the friction for the hard side so the marketplace can grow.

## Growth Dynamics

- **Local density matters**: A landscaping network in Austin doesn't help a restaurant in Houston. Moil must achieve local density in each market before expanding.
- **Reputation loops**: Workers talk. If Moil placements lead to good jobs, workers refer each other → organic supply growth.
- **Employer anchor accounts**: A warehouse with 100 workers is a better anchor than 10 restaurants with 5 workers each. Anchor employers pull supply to the platform.

## Key Risks

- **Disintermediation**: Once employer and worker connect, do they need Moil? Mitigation: value-add at the hiring stage (screening, matching) + value-add after (retention tracking, re-hire).
- **Race to the bottom on price**: If competitors subsidize one side, Moil can't match. Defense: differentiation on quality (voice-first, Spanish-first) not price.
