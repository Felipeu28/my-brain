## Summary

This call was scheduled as an AI Foundry advisory/discovery session with Paula Florez-Estrada Valverde from Microsoft. It pivoted almost entirely to an Azure account crisis that Andres disclosed — the entire Moil Azure directory had disappeared after a Google Workspace → Microsoft 365 migration, taking down servers and subscriptions for ~1 week.

## The Azure Crisis (May 2025)

**What happened:**
- Moil was migrating from Google Workspace to Microsoft 365
- After the migration, the entire Azure account directory disappeared
- All subscriptions (including Azure Foundry sponsorship credits) became inaccessible
- Servers went down — team couldn't access or modify the production website for ~1 week
- Andres had been speaking to Microsoft support, but support blamed the migration; no resolution yet

**Impact:** All Azure activity blocked. AI Foundry exploration blocked. Production website at risk.

**Paula's response:**
- Will look into the issue; asked for the Azure support ticket number to escalate on her end
- Offered to send AI Foundry quick starts and resource links to continue planning
- Recommended: submit/escalate the open ticket with the exact tracking ID

**Quote from Andres:** *"It literally disappeared and we don't know why... we haven't been able to do anything with our website for about a week."*

## AI Foundry Discussion (Limited — blocked by crisis)

Paula was introducing the **Azure AI Foundry** as the platform for Moil to build AI features using their Microsoft Founders Hub Azure credits. Key notes:
- AI Foundry should use the Azure sponsorship subscription (not create new subscriptions)
- If the Foundry asks to create a subscription, use the dropdown to select existing one
- Once the Azure account is restored, this is the recommended path for Moil's AI development

**Andres's AI vision as stated:** Moil runs fully on AI. Uses Azure credits for ~40% research/learning, ~60% running the business. Goal: become an "AI-first company."
