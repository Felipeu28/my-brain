## Summary
A meta-guide describing Andres's personal "Brain" system: a five-layer pipeline that turns weekly X bookmarks and other raw signal into a compounding personal knowledge graph queried through Claude. It is the architectural blueprint this knowledge base implements.

## Key Points
- **5 layers:** Collection (X bookmarks) → Storage (`raw/`) → Processing (you) → LLM Wiki (synthesis) → Query (decisions).
- **Themes & Takeaways** summaries per digest = compressed intellectual history over time.
- **Cadence:** daily 10 min skim, weekly 30 min processing, monthly 20 min synthesis, quarterly deep query.
- **Automation:** weekly X bookmark sweep via Claude Cowork scheduled task (Mondays 9 AM).
- **Future sources to add:** email digests, Slack, GitHub activity, reading list (Readwise/Pinboard).
- **MOIL.md context file:** Mission, Operating Principles, Identity, Legacy — primer for any big Claude query so output sounds like Andres, not generic advice.
- **3 query patterns:** (1) Synthesis ("what am I obsessed with?"), (2) Decision ("A vs B — what do my notes say?"), (3) Deep Dive ("teach me everything I know about X").
- **12-month roadmap:** automation → multi-stream → first synthesis → topic skills → product feedback → queryable decision engine.
- **Core truth:** "Collection is automatic. Processing is the discipline. Synthesis is the payoff."
